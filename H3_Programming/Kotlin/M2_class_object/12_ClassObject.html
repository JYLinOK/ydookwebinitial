类 class
类是面向对象中 类与对象 两巨头中巨头之一。
类也是面向对象的编程中的主要整体聚合的数据结构。

Kotlin 中的类使用关键字 class 进行声明定义。
Kotlin 中的类的声明定义由：类名、类头及类内容构成。

类名: 类名就是类的数据结构的名称，用于类的实体化。
类头: 类头是类定义的开头部分，包括指定类的类型参数、类的主构造函数等。
类内容: 类内容就是用花括号括起来的类声明定义的具体部分。


class 关键字
Kotlin 使用 class 关键字作 类 的声明定义。

类的内容主体使用花括号 { } 括起来。

1 
class MyClass { ... }



空类
如果一个类没有主体，那么该类就是空类。
空类不需要使用花括号定义内容。

1 
class Empty




构造函数
每次使用类来实体化新建对象的时候，都需要执行一次构造函数。
构造函数就是类内部的新建对象时的初始化执行函数。

在 Kotlin 中，可以为每一个类设置一个 主构造函数 及一个或者多个 次构造函数。

主构造函数是类头的一部分。声明主构造函数的时候，应该在类名后面进行声明定义。
如果类名后面有可选的类型参数，则主构函数应该在可选参数的后面进行声明定义。

如果声明类的时候不定义构造函数，Kotlin 会为类默认分配一个空构造函数。



主构造函数
主构造函数是 Kotlin 类定义中的首要构造函数。

主构函数内不能包含任何执行代码。初始化代码需要放在以 init 关键字作为前缀的 初始化块 中。
主构函数内可以设计一个或多个 init初始化块，也可以设计 0个 init初始化块。

init初始化块在进行初始化时的执行顺序是根据设计的书写顺序从上到下执行。



constructor 关键字
Kotlin 的主构造函数通过关键字 constructor 进行定义。

1 
class Person constructor(ID: String) { ... }



省略 constructor 关键字的主构造函数定义
如果主构造函数中没有任何注解或者可见性修饰符，可以省略 constructor 关键字。
省略 constructor 关键字后的主构造函数的声明定义语法功能及属性不变。

1 
class Person(ID: String) { ... }



初始化块（initializer blocks）
在 Kotlin 中定义主构造函数的时候，不能直接定义初始化执行代码。

如果需要定义初始化执行代码，需要在定义主构造函数内部的，以 init 关键字作为前缀的 初始化块 （initializer blocks）中。

设计好主构造类后，在该类进行实例初始化期间，主构造类内的初始化块按照它们在主构造类体中的定义顺序依次执行。

主构造类的初始化块在类初始化的时候，与属性初始化器交织在一起进行初始化。

    val a_parameter = "first parameter"

初始化中调用主构造函数的参数及方法
在 Kotlin 中定义主构造函数的时候可以进行。

1
2
3 
class MyClass(attribute:String){
    val a = attribute.functionA()
}



初始化主构造函数的属性声明的简化
初始化主构造函数的属性声明可以在类声明的类名后面的括号内直接定义。

简化的主构造函数的属性声明可以在圆括号内定义使用 var 定义变量与使用 val 定义常量。

1 
class MyClass(val a: String, val b: Int, var c: Int) { …… }



次构造函数
次构造函数是 Kotlin 类定义中的辅助构造函数。
类的次构造函数通过 constructor 关键字在类的内容内部进行定义。

代码示例：

1
2
3
4
5
6 
class MyClass {
    var something: MutableList<BigClass> = mutableListOf<BigClass>();
    constructor(parent: BigClass) {
        parent.something.add(this)
    }
}



次构造函数委托主构造函数
次构造函数可以直接委托主构造函数。

次构造函数也可以通过通过别的次构造函数间接委托主构造函数。

如果次构造函数委托到同一个类的主构造函数，使用 this 关键字进行委托。

代码示例：

1
2
3
4
5
6
7 
class MyClass(val name: String) {
    var something: MutableList<BigClass> = mutableListOf<BigClass>();
    
    constructor(name: String, parent: BigClass): this(name) {
        parent.something.add(this)
    }
}
