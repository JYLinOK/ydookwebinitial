容器的数据处理理念

容器是一种用于存放多个数据量的数据类型。
在编程语言中，容器时编程者重要的处理数据类型。
容器在编程者处理复杂多组分数据的时候十分有用。





Kotlin 的基本容器类

Kotlin 有 3 种容器类，分别为：
1. 集合 Set ；
2. 队列 List ；
3. 映射 Map 。

由于 Kotlin 中的每种容器都可以分为 只读容器 与 可变容器 两种，所以 Kotlin 共有 6 种类型的容器。

Kotlin 新建容器时，默认新建的是 只读容器 。

下面是具体分类：

container type container name container initialize
read-only set Set setOf()
mutable set mutableSet mutableSetOf()
read-only list List listOf()
mutable list mutableList mutableListOf()
read-only map Map listOf()
mutable map mutableMap mutableMaptOf()






Set / MutableSet  集合

Set 集合是 Kotlin 中最简单的容器。


Kotlin 的集合具有以下性质：
1. 集合内部的元素具有无序性，无法通过指定下表的形式访问处于具体位置的集合元素。
2. 集合内部的元素具有唯一性，Kotlin 在新建集合的时候，通过哈希值校验是否已经存在相同的元素，如果存在，则覆盖原有元素。


Set 只读集合（不可变集合）具有以下性质：
1. Set 集合一经确定，就不能再修改。
2. 如果需要进行后期的元素变更，不能定义新建 Set 集合，而是应该定义新建 MutableSet 集合。


MutableSet 可变集合具有以下性质：
1. MutableSet 集合的操作是无位置性的。也就是说，不能对 MutableSet 集合内的指定位置进行修改。
2. MutableSet 集合在进行元素的增加、删除、更改或者查找时，无法对某一个具体的位置的元素进行操作。
3. 虽然 MutableSet 集合具有无顺序性，但是我们仍可以对 MutableSet 集合执行遍历操作。


MutableSet 可变集合支持的遍历操作：
1. for-in 循环遍历；
2. for-each 循环遍历；
3. 迭代器遍历。


for-in 循环遍历

迭代器
迭代器与指针类似，迭代器本身不代表元素，迭代器指向的是元素的存放地址。

迭代器遍历的实质就是遍历所有元素的地址。

迭代器通过 .hasNext() 方法判断是否存在下一节点。如果存在下一节点，则继续遍历。如果不存在下一节点，则遍历完成。

迭代器通过 .next() 获取 MutableSet 可变集合对应下一节点的元素。

下来代码展示的是使用 迭代器 循环遍历 MutableSet 可变集合：

1
2
3
4
5 
var alist:List<String> = listOf("item1", "itme2", "itme3", "itme4", "itme5")

while (alist.hasNext()){
    print(alist.next()) 
}






List / MutableList  队列

之所以将 List / MutableList 翻译成 队列 而不是 列表，就是为了强调 List / MutableList 的有序性。
也就是说，List / MutableList 是有顺序的。List / MutableList 中的元素可以通过顺序下标进行访问。


Kotlin 队列具有下列性质
1. 队列具有顺序性。队列中的元素排列具有顺序性。
2. 队列可以通过 .get() 方法来获取队列中指定位置的元素。
3. 队列中的元素可以直接通过下表进行获取访问。
4. MutableList 是可变队列，可以进行 增删查改 操作，List 是只读队列，只能进行 查 操作。
5. Mutablelist 中的 .add() 方法可以将新增元素添加到原有队列的末尾，也可以将新增元素添加到队列中的指定位置。
6. Mutablelist 中的 .set() 方法更新修改队列中指定位置的元素。
7. Mutablelist 中的 .removeAt() 方法删除队列中指定位置的元素。


MutableList 可变集合支持的遍历操作：
1. for-in 循环遍历；
2. for-each 循环遍历；
3. 迭代器遍历。
4. 按照下标遍历。


MutableList 按照下标遍历：
1. MutableList 的下标存放在下标数组 indices 中。可以通过 MutableList 直接调用进行使用。
2. 通过遍历 .indices 数组就可以遍历对应的队列的下标。

下面是通过 .indices 下标遍历来遍历队列：

1
2
3
4
5 
var alist:List<String> = listOf("item1", "itme2", "itme3", "itme4", "itme5")

for(i in alist.indices){
    print(alist[i]) 
}






Map / MutableMap  映射

Kotlin Map / MutableMap  映射 的本质就是 键值对 。
键值对就是 索引 + 内容 的组合体。
Map 是只读映射，不能编辑修改内容。MutableMap 是可变映射，可以编辑修改内容。



Kotlin 映射具有下列性质
1. 映射的索引与内容是一一对应的关系。
2. 映射中的每个元素都由 键名 与 键值 两部分组成。
3. 映射中的键名具有唯一性，不同元素的键名不能相同。


MutableList 可变集合支持的遍历操作：
1. for-in 循环遍历；
2. for-each 循环遍历；
3. 迭代器遍历。
