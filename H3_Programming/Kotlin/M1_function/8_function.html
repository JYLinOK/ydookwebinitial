函数的传入参数

Kotlin 传入函数参数语法
Kotlin 函数传入参数定义使用的是 Pascal 表示法。
Pascal 表示法的基本表示方式为: name: type。

如果圆括号内具有多个传入参数，需要使用 , 号隔开。

1
2
3 
fun function_name(x:Int, y:Int) : Int {
    return x*y       
}


Kotlin 函数 传入参数 默认参数
Kotlin 在函数传入函数参数定义的时候可以定义参数的默认值。

Kotlin 可以在函数输入圆括号内直接使用 = 号赋值函数参数的默认值。

默认参数的功能是，如果被定义了默认参数的函数被调用的时候没有输入相应的参数，被调用函数将自动调用函数对应的原先定义的默认参数。

1
2
3 
fun function_name(x:Int = 1, y:Int = a.number1) : Int {
    return x*y       
}

上述程序实现了 1 与 a.number1 的整数相乘运算。


Kotlin 调用函数时默认参数的可省略性
Kotlin 在进行具有默认参数的声明后，调用默认参数时候可以进行省略。也就是默认参数在函数被调用时可以省略不赋值。

如果函数的传入默认参数没有在函数被调用的时候被赋值，则 Kotlin 会在被调用的函数执行的时候自动提取默认参数的默认值使用。

分析上面结果，可以得到以下结论：
1. Kotlin 函数在被调用的时候，命名参数的位置并不强制。命名参数的书写位置可以互换。但是需要满足一定的条件，以免造成参数混淆。

2. Kotlin 函数在被调用的时候，函数对应的命名参数可以不用写全，只是书写部分参数。

3. Kotlin 函数在被调用的时候，函数对应的命名参数方式对默认参数进行赋值，从而赋值默认参数新的值。但是默认参数的原有默认值不会改变。

4. 由上述结果的26、28与30行得知，Kotlin 在进行函数调用的时候，命名参数后面不能出现不带参数名的默认参数的值。

5. 带有默认参数的函数在被调用的时候，有哪一部分的默认参数没有被复制定义，其对应的默认值就会被默认调用。

Kotlin 函数 传入参数 命名参数
参数是带参数函数的重要组成部分。

Kotlin 的命名参数是为了增加函数调用时的参数描述的可读性相对于 Java 新增的一种函数参数描述方式。

使用命名参数描述函数参数的方式是：使用 参数名 = 参数具体内容 在函数的圆括号内进行描述。

使用命名参数方式定义函数后，在调用函数时，函数圆括号内的参数的 书写顺序 可以跟原来函数声明的时候不同。

命名参数书写方式可以与默认参数书写方式结合使用。

1
2
3
4
5 
fun TAdd(a: Int = a.number1, b: Int = a.number2, c: Int = a.number3) : Int { 
    return (a+b+c)
}

TAdd(c = 1, b = 3, a = 6)
