各种图搜索的基本策略

下面内容将对AI领域的各种图搜索的基本策略进行简要介绍。


A. 基本概念


⚓  1. 什么是图搜索？

✪ 图搜索是一种搜索方法策略的表现模式，也可称为体现方式。与图搜索对应的表现形式是树搜索。



⚓  2. 对于一个搜索问题，应该用树搜索还是图搜索？

要解决这个选择问题，我们首先得知道图搜索与树搜索的特点区别：
1. 图搜索结构中，不允许搜索图出现相同的搜索节点；
2. 树搜索结构中，允许搜索图出现相同的搜索节点；

以及各自的优缺点：

1. 树搜索的优点是：控制简单；

2. 树搜索的缺点是：占有的空间大，算法节点多，需要较大的时间及空间来实现算法。

3. 图搜索的优点是：可以节省大量的存储节点的空间，相同的算法节点只是记录存储一次。 可以节省大量的用于存储节点的时间，相同的算法节点不需要进行重复产生。

4. 图搜索的缺点是： 每产生一个新的算法节点，都要判断该节点是否已经生成过，因而在算法控制比较复杂。 另外判断是否是已经产生的算法节点也需要耗费算法的执行时间。


✪ 在选择使用图搜索还是树搜索的时候需要权衡利弊后再选择最合适的算法表现形式。





B. 图搜索算法的特点

✪ 图搜索通用算法的特点：

1. 图搜索中，只记录状态空间中那些已经被搜索过的状态，这些搜索过的状态用一个搜索图 G 来表示。
2. 搜索图 G 由两张存放节点的 表 组成：
   2.1 Open 表：用于存放已经生成过的，而且是已经用启发式评价函数评价与测试过的， 但是又还没有产生后续节点的算法节点，这些节点也被称为考察节点；
   2.2 Closed 表：用于存放已经生成过的，而且是已经产生了后续节点的算法节点。
3. 图搜索还有一个辅助算法结构：Tree : 该结构存放的节点是 搜索图 G 的一个子集。
4. Tree 用来存放已经生成的搜索树，这些搜索树由 搜索图 G 的反向边(也即反向指针)组成。





C. 或图搜索特点

⚓ 1. 通用或图搜索的搜索算法

设：S0为搜索的初始状态，Sg为搜索的目标状态，搜索的或图标记为G。
1. 首先产生一个仅由S0组成的Open表 => Open = (S0) ;
2. 然后产生一个空的Closed表;
3. 判断 => 如果产生的Open表为空，则本次搜素失败，退出搜素；
4. 操作 => 在Open表中按照指定的规则选择出下一个节点，将该节点设为n;
5. 操作 => 将节点n放至Closed表中，从Open表中去掉节点n;
6. 判断 => 如果节点n属于目标节点状态集合，即n∈Sg,则搜索成功，退出搜索。 此时的搜索成功解为在搜素树中。沿着指针从节点n指向S0的路径，即是从最终的满意搜索状态往回指向最初的搜索状态。
7. 操作 => 产生节点n的一切后继，将后继中不是节点n的前驱点的所有点构成的集合M装入或图G中,并此集合M中的节点作为节点n的新的后继；
8. 对M中的元素P节点进行两类处理：
  a. 判断 => 如果 P∉G : 即是 ：P既不在Open表中也不在Closed表中 =>
  处理：根据制定的原则将P同时加入到Open表及Closed表中。对P进行评估，再放入搜索树中。
  b. 判断 => 如果 P∈G : 是否需要改变搜素树中的P指向（到）节点n中的指针。
9. 转至第 3 步骤。



⚓ 2. A*算法

设：S0为搜索的初始状态，Sg为搜索的目标状态，搜索的或图标记为G。
1. 首先产生一个仅由S0组成的Open表 => Open = {S0} ;
2. 然后产生一个空的Closed表 => Closed ={} ;
3. 判断 => 如果产生的Open表为空，则本次搜素失败，退出搜素；
4. 操作 => 在Open表中按照指定的规则选择出f(n)值最小的节点，将该节点设为n;
5. 操作 => 将节点n放至Closed表中，即有：
   fn(n) = g(n) + h(n), h≤h*
6. 判断 => 如果节点n属于目标节点状态集合，即n∈Sg,则搜索成功，退出搜索。 此时的搜索成功解为在搜素树中。沿着指针从节点n指向S0的路径，即是从最终的满意搜索状态往回指向最初的搜索状态。
7. 操作 => 产生节点n的一切后继，将后继中不是节点n的前驱点的所有点构成的集合M装入或图G中,并此集合M中的节点作为节点n的新的后继；
8. 对M中的元素P节点进行两类处理：
  a. 判断 => 如果 P∉G : 即是 ：P既不在Open表中也不在Closed表中 =>
  处理：根据制定的原则将P同时加入到Open表中。对P进行评估，再放入搜索树Tree与G中。
  b. 判断 => 如果 P∈G : 是否需要改变搜素树中的P指向（到）节点n中的指针。并且更改P的子节点n的指针及费用。



⚓ 3. A*算法的特点

相对于一般的最佳优先算法而言，A*算法的独特之处在于：
1. 如果问题有解，A*算法一定可以找到最优解：S0 -> Sg 一定行得通。
上述特性称为 可采纳性 。
