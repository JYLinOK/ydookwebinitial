对象是 Kotlin 数据类型的核心
在 Kotlin 的世界中，所有数据类型都可以看作是对象。

按照 Kotlin 面向对象的编程理念，使用者可以给任意变量定义成员函数及属性参数，也可以调用任意变量已经定义的成员函数或属性参数。

Kotlin 的基本数据类型有：

1. 数字 ；
2. 字符 ；
3. 布尔值 ；
4. 字符串 ；

字面常量 : 给数据类型变量赋值

使用数据类型定义的，具有具体的赋值的数据常量称为字面常量。


常量一般有两种表示方式：

1. 字面常量：
直接使用 字面形式 表示的常量，也称为直接常量。
字面形式指的是使用字符串直接表示数据的数字量。比如：0，1，100 等。
简单地讲，字面常量就是用数字文字（不限于阿拉伯数字：十进制）表示的数值常量。


2. 符号常量：
使用 标识符 表示的常量。
符号常量通过标识符进行定义，比如：C语言中的：#define 标识符。

比如 C 语言中的下面语句就是将标识符 A 定义为常数常量 123 :
#define A 123

通过上述定义后，在整个源文件中都可以使用 A 代表数值 123 。


Kotlin 支持的数值字面常量有：
1. 二进制
2. 十进制
3. 十六进制

注意： Kotlin 不支持八进制

Kotlin 数据类型转换

一般来说，编程语言的数据类型转化都有两种：
1. 隐式转换：编程者不用进行额外语句编写操作，定义的数据类型可以自动转换。转化的数据类型对应的数值范围应该是由大到小。

2. 显式转换：当定义的数据类型不能自动转换时，编程者需要进行额外语句编写操作，转化的数据类型对应的数值范围应该是由小到大。

隐式转换也称为自动转换，显示转换也称为强制转换。


Kotlin 隐式转换
Kotlin 可以默认进行隐式转换。

在 Kotlin 中，String 类型属于一种特殊的基本数据类型。

在 Kotlin 中的数组类型中，不包括字符串数组类型。

在 Kotlin 中，不能直接定义 String 类型的数组，要想定义 String 类型数组，需要使用 Array<String > 类型，同时分配字符串数组的方法变成 arrayOf 。

Kotlin 定义字符串数组的方法如下:
