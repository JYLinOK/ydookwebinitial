1. Kotlin 包 package 声明

Kotlin 源文件通常以 package 声明为开头。

在源文件的开头声明了 package 之后，该源文件内的所有函数及类都被定义包含于这个 package 中。


下面是 Kotlin package 相关程序代码：

1
2
3
4
5

8
9 
package my.package

fun function1() {...}

fun function2() {...}

Class Class1() {...}

//...


上述代码中：

1. function1() 的全名是 my.package.function1 , 该函数属于包 my.package 中。

2. function2() 的全名是 my.package.function2 , 该函数属于包 my.package 中。

3. Class1 的全名是 my.package.Class1 , 该类属于包 my.package 中。





2. Kotlin 默认 package

如果Kotlin源文件中没有声明 package ，那么该源文件该默认引入的是无名字的 默认包 。

下面是：Kotlin 源文件在没有声明 package 的时候自动导入的一些 package ：

1. kotlin.*
2. kotlin.annotation.*
3. kotlin.collections.*
4. kotlin.comparisons.*
5. kotlin.ranges.*
6. kotlin.sequences.*
7. kotlin.text.*


此外，Kotlin 编译器会根据源文件的工作平台的不同导入其他对应的包：

JVM：
1. java.lang.*
2. kotlin.jvm.*


JVM：
1. kotlin.js.*





3. Kotlin package 的导入

导入 package 时使用关键字 import 。

导入 package 的方式有：

1. 使用 . 号进行直接调用，直接导入 package 中的一个名字：

1 
import p1.package1


2. 使用 * 号代表全部内容，导入一个作用域下（某个 package 中）的所有内容（包括：包、类、对象等）:

1 
import p1.*



package 导入重命名
当被导入的 package 的名称太长，需要进行包名称简化。或者为了避免包名称重复造成冲突，此时可以进行包的导入重命名。

package 导入重命名，主要通过关键字 as 实现：

1
2
3 
import pak.LongLongNamePackage  // type LongLongNamePackage can be called 

import p1.LongLongLongNamePackage as LLLP  // type LongLongLongNamePackage can be called as LLLP




3. import 不仅仅可以用来导入 package 中的类

使用关键字 import 也可以导入其他声明，比如：

1. 顶层函数机器属性；
2. 在对象生命中声明的函数及属性；
3. 枚举常量。

与 Java 不同的是，在 Kotlin 中，没有单独使用的 “import static” 导入语法。所有这些声明都用 import 关键字导入。

也就是说，Kotlin 只使用 import 关键字进行 package 内容的导入。
