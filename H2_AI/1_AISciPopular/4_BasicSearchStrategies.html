解决搜索问题的基本策略

下面内容将对AI领域的用来解决搜索问题的常用基本策略进行简要介绍。


A. 盲目搜索算法方法   Uninformed Search


⚓  1. 宽度优先算法

✪ 宽度优先算法指的是：在一个搜索树中，按同层的邻近节点依次展开搜索节点的搜索策略算法。

一般来说，盲目搜索算法存放搜索节点的方式都是通过 表 来实现。
表 是一种简单的数据结构。在 表 的首部及尾部都可以增加或者删除搜索节点。


✪ 一般来说，宽度优秀算法的执行过程如下：
1. 确定初始状态，令 N 为一个由搜索的初始状态构成的表；
2. 判断：如果 N 则退出搜索，并定义此次搜索策略是失败的；
3. 令 n 为 N 的第一个搜索节点，然后将 n 从 N 中删除；
4. 若 n 是目的节点，则退出当前搜索，并定义此次搜索是成功的；
5. 若 n 不是目的节点，则将 n 节点的后续节点继续加入到 n 节点的末端，重现开始第二步， 直到得到的搜索节点为目的节点。

盲目搜索是一种直到型搜索，意思是：如果某个搜索问题的最优解存在，那么盲目搜索将一直搜索，直到找到这个最优解为止。

当然也有可能是：该搜索问题并不存在最优解。这得对整个搜索问题完成一次盲目搜索后才能得到。

✪ 所以，盲目搜索的优缺点容易总结为：
1. 如果某个搜索问题存在最优解，那么盲目搜索必定会找到该最优解。(因为盲目搜索是对全部解的节点都进行搜索。)
2. 既然是全部搜索，相对于部分搜索而言，盲目搜索耗费的搜索时间势必会更长。这就造成了其搜索效率较低。
尤其是对一些具有组合爆炸的搜索问题，盲目搜索策略显然并不适合。





⚓  2. 深度优先算法

✪ 深度优先算法是一种与宽度优先算法对应的搜索算法。

很显然，顾名思义我们也能够发现，深度优先算法强调的是搜索算法的深度，而宽度优先算法强调的是搜索算法的宽度。

那么我们怎样来理解搜索的算法的宽度与深度呢？

宽度： 我们可以理解为搜索算法的体面，强调的是搜索算法的全面性。强调的是对应同一层的搜索节点的全面覆盖的程度。
深度： 我们可以理解为搜索算法的位面，强调的是搜索算法的层次性。强调的是对应不同层的搜索节点的层次挖掘的程度。

在深度优先算法中，最深的节点，也就是最新产生的节点会被优先拓展到表中。
在深度优先算法中，深度相等的搜索节点的排布顺序不定，可以任意进行排列。


✪ 一般来说，宽度优秀算法的执行过程如下：
1. 确定初始状态，令 N 为一个由搜索的初始状态构成的表；
2. 判断：如果 N 则退出搜索，并定义此次搜索策略是失败的；
3. 令 n 为 N 的第一个搜索节点，然后将 n 从 N 中删除；
4. 若 n 是目的节点，则退出当前搜索，并定义此次搜索是成功的；
5. 若 n 不是目的节点，则将 n 节点的后续节点继续加入到 n 节点的首端，重现开始第二步， 直到得到的搜索节点为目的节点。

✪ 通过对比可以知道，与宽度优先算法的执行步骤相比，深度优先算法的唯一不同点就是步骤中的第5步的“末端”变成了“首端”。

✪ 所以，深度优先搜索的优缺点容易总结为：
1. 深度优先搜索只是针对某一个或者几个特定的搜索方向进行搜索，所以深度优先算法的执行效率比较高。
2. 深度优先算法的搜索速率及效率虽然高，但是由于搜索位面的数量不全，很可能发生的情况就是， 执行了一次深度优先算法后并没有找到最优解或者理想的解。
3. 深度优先算法还有另外一种不好的搜索结果是，如果原搜索问题是深度无限的搜索，那么深度优先算法就很可能一直无限执行下去。 这显然也不是我们期望的结果。

✪ 深度优先算法与宽度优先算法的比较：
1. 一个是侧重宽度，一个是侧重深度；
2. 两者在搜索侧重上不同，但是在搜索策略上很相近；
3. 一般两者的假设都是以队列作为数据结构，对每次选择的队列的第一个节点进行拓展；
4. 宽度优先搜索会将每次子搜索的结果放在队列的尾部，而深度优先搜索会将每次子搜索的结果放在队列的首部。





⚓  3. 分支有界搜索算法

✪ 分支有界搜索算法属于深度优先算法的一种。特点是分支有界算法对每一个搜索分支都规定了同一个搜索深度， 即是各个不同的搜索分支都采用统一的搜索深度。

也就是说，分支有界搜索算法实在原来的深度优先算法的基础上，为某些搜索分支设置了相同的搜索深度界限。
这个搜索界限是非常有用的，它可以限制深度优先算法进行无限深度的搜索。

也就是说，当分支有界搜索算法沿着搜索树对某一搜索分支进行搜索后， 达到了限定的搜索界限仍然没有得到理想的搜索节点(即搜索结果)，那么分支有界搜索算法将会停止对该搜索分支的继续搜索， 转到对其他搜索分支进行深度搜索，如此反复，直到得到理想的搜索结果或者满足退出条件。


✪ 分支有界算法的一般执行步骤如下：
1. 确定初始状态，令 N 为一个由搜索的初始状态构成的表；
2. 判断：如果 N 则退出搜索，并定义此次搜索策略是失败的；
3. 令 n 为 N 的第一个搜索节点，然后将 n 从 N 中删除；
4. 若 n 是目的节点，则退出当前搜索，并定义此次搜索是成功的；
5. 若 n 搜索节点的深度已经达到了算法设定的最大搜索深度界限 Dmax , 则返回步骤2；
6. 若 n 搜索节点的深度还没有达到算法设定的最大搜索深度界限 Dmax , 则将 n 的后续搜索节点加入到本搜索表 N 的首部，再返回步骤2；
7. 一直执行，得到理想结果或者满足条件退出搜索。

一般而言，分支有界算法的搜索深度都会浅于搜索树的深度，所以分支有界算法的搜索效率一般都比普通的深度优先算法的要高。





⚓  4. 迭代加深搜索算法

迭代加深搜索算法是在分支有界搜索算法的基础上加工得到的。

迭代加深搜索算法是一种兼顾搜索深度与搜索宽度的搜索算法。

迭代加深搜索算法的加工特点是：对分支有界搜索算法的最大搜索深度界限 Dmax 进行迭代处理，然后逐步增加搜索深度。

✪ 迭代加深搜索算法的搜索结果特点是：在有限的搜索深度内，保证了对一部分搜索节点进行宽度搜索， 同时，如果对同一层的搜索节点的宽度搜索没有找到理想的搜索结果，算法就会增加 Dmax ， 然后继续展开深度搜索与宽度搜索。一直执行，得到理想结果或者满足条件退出搜索。





B. 启发式搜索算法方法   Heuristically Search
启发式搜索算法是针对盲目搜索的低效率提出的改进搜索算法。也称为 有信息搜索 Informed Search 。

⚓  1. 启发式搜索算法的特点

✪ 启发式搜索算法在进行搜索时，不再是像盲目搜索一样，任取一个搜索分支然后一直进行搜索。 而是在搜索的过程中，根据一些被提供的启发式信息，不断地调整与确定最合适的搜索方向，选择最合适的搜索分支进行下一层次的搜索。





⚓  2. 启发式搜索的理论依据

启发式搜索的理论依据主要有以下3点：

1. 人们在解决搜索问题的过程中，可以根据一些启发式信息来临时改变搜索的方向或已经确定的策略， 以或者最佳的搜索结果；

2. 基于实际问题需要的搜索问题，一般只要找到一个满足条件的搜索结果即可。一般情况下， 没有必要将全部满足条件的搜索解都找出来。所以一般我们不需要采用盲目搜索来寻找所有合适的解。

3. 使用启发式搜索，有了一些启发式信息的辅助及指引，整个搜索会变得更加具有目的性。

4. 启发式搜索可以避免在某些搜索问题领域可能遇到的组合爆炸的问题。





⚓  3. 启发式搜索的方向选择

1. 正向搜索：从初始状态出发向着目的状态进行搜索；
2. 逆向搜索：从目的状态出发向着初始状态进行搜索。
3. 双向搜索：将正向搜索与逆向搜索结合起来，同时从初始状态与目的状态出发，朝着相对的方向进行搜索的算法就叫双向搜索。

使用双向搜索一般是用来提高搜索效率的。理想的双向搜索直达目的结果，但是不理想的双向搜索可能花费的时间要比单独使用正向搜索或者是逆向搜索的要多， 或者说效率要低。

那么，在选择搜索算法的时候，我们是选择正向搜索还是逆向搜索呢？

可以采用以下方法进行判断选择：
1. 沿着分支因子低的方向进行搜索会更加有效率；
2. 沿着状态少的搜索方向进行搜索会更加有效率。





⚓  4. 几种最基本的搜索策略

1. 生成测试法:
(1)、 首先生成一个可能解，该解可以是状态空间的一个点，也可以是一条从 S0 出发的搜索路径。
(2)、 对比生成解与目的解的差距；
(3)、 如果生成解与目的解的差距很小，满足设定的条件要求，则称为找到了目的解，结束搜索，否则返回第一步继续生成解。



2. 爬山搜索法:

此法也成 瞎子爬山法 。
此法特点是： 搜索过程中只是使用局部的反馈信息，好像就是瞎子探路爬山巡山一般。

✪ 此法执行步骤如下：
1. 首先生成一个生成解，判断：如果该解满足目的解的条件，则称已经获取目的解，搜索完成。否则执行第二步。
2. 从当前解出发，继续生成生成解，继续进行判断。
   2.1 使用测试函数测试新的生成解，如果满足条件，则停止搜索并称获得最优解。否则执行下一步；
   2.2 将新生成的解与已经通过测试函数验证的解进行比较，如果新生成的解更加接近目的解， 则将该解保留作为最佳元素。
   否则将该新生成解进行舍去。
3. 以当前的最佳元素作为搜索起点，转至第二步。
✪ 需要注意的是：
爬山搜索法最终获得的最优解一般来说局部最优解。



3. 最佳优先搜索法:
此法源于爬山搜索法，并对爬山搜索法进行了改进。

✪ 此法执行步骤如下：
1. 首先生成一个生成解，判断：如果该解满足目的解的条件，则称已经获取目的解，搜索完成。否则执行第二步。
2. 从当前解出发，继续生成生成解，继续进行判断。
   2.1 使用测试函数测试新的生成解，如果满足条件，则停止搜索并称获得最优解。否则执行下一步；
   2.2 将新生成的解生成的解添加到原可能解的集合中，形成新的可能解集合。
3. 以当前的最佳元素作为搜索起点，转至第二步。





C. 随机搜索算法方法   Heuristically Search

⚓  1. 模拟退火算法

✪ 此法是克服瞎子爬山法的缺点的有效搜索算法。


✪ 模拟退火算法的基本思想是:

在问题系统中，如果整体的系统趋势是沿着能量降低的方向运动的， 则偶尔允许系统状态跳转到能量较高的状态，从而避开局部极小点，最终稳定地达到全局最小点。





⚓  2. 遗传算法

✪ 遗传算法的基本思想源自于达尔文的进化论。

✪ 遗传算法是一种仿生算法，模仿的是生物界的自然选择与遗传机制。

✪ 遗传算法适用于复杂的非线性问题。

✪ 遗传算法的基本执行步骤是：
1. 编码；
2. 生成初始种群；
3. 计算适应度；
4. 算法选择；
5. 算法交叉；
6. 算法变异。

✪ 遗传算法只是使用目标函数(也称为适应值)进行搜索，可以解决几乎所有的搜索问题。





⚓  3. 人工免疫算法

此法的基本设计思想是：

模拟人体的免疫系统的免疫细胞的工作机制进行设计。


此法与遗传算法类似，但是在把遗传算法中的遗传因子以及控制参数进行了调整及改变。





⚓  4. 蚁群算法

此法的基本设计思想是：

模拟蚂蚁从巢穴出发，通过跟踪信息素(pheromone)来寻找及改变出发到食物之间的最短路径的算法。

此法在模拟蚂蚁觅食的过程中需要进行以下假设：
1. 每一只蚂蚁的运动都是随机的；
2. 信息素被洒在蚂蚁经过的路径上；
3. 每一只蚂蚁都能够感知周围的信息素；
4. 如果某条前进觅食路径上的信息素的浓度越高，则证明该条路径被蚂蚁选择的可能性就越大。

此法在执行的过程中受到以下6个规则的控制：
1. 范围；
2. 环境；
3. 觅食规则；
4. 移动规则；
5. 避障规则；
6. 播撒信息素的规则。

随机搜索是此法的搜素特点之一。





⚓  5. 粒子群算法

粒子群算法也称为 PSO 算法，英文为 Particle Swarm Optimization 。

粒子群算法同时具有蚁群算法与遗传算法的特点：
1. 粒子群算法与蚁群算法都是采用增量的方式进行搜索；
2. 粒子群算法在算法结构上的种群初始化、适应性函数、终止条件基本保持与遗传算法一致。
3. 粒子群算法中的“粒子”相当于遗传算法中的个体，但是不同的地方在于： 粒子群算法中设置了一个速度的概念，以决定粒子的飞翔方向与距离；
4. 粒子群算法与遗传算法都是通过迭代找到最优解。


✪ 粒子群算法在每一次迭代中，会通过两个“极值”来更新算法本身：
1. 第一极值：算法本身找到的最优解；
2. 第二极值： 在整个种群中找到的最优解。

粒子群算法的第二极值也就是全局最优解。该解也被称为“个体极值 pBest”。





⚓  6. 随机混合算法

不同的随机算法都有自身的算法缺点。一般我们可以通过两种方式来弥补这些缺点：
1. 通过提高算法本身来减少缺点：比如调整算法的控制机制、优化算法的设置参数等；
2. 通过混合其他算法的优势来优化自身算法的缺点。

下面介绍两种常用的算法混合组合：
1. 模拟退火算法 + 遗传算法 + PSO算法；
2. 蚁群算法 + 遗传算法 + PSO算法。
