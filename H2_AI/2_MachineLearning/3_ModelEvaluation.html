模型评估   训练与选择合适的模型

1. 评价学习模型优劣的标准


我们可以通过合理的设计及丰富的数据训练春大量的模型，但是，由于设计思想及算法思想或者训练数据集调控参数的不同，我们很可能会得到性能不一的各类模型。

作为一门强调性能及效率的科学，我们希望获得的学习算法是高效而强大的。既然如此，我们就要对不同的学习算法进行评估比较，以选择最合适的算法或者学习模型。


1.1 误差

误差用于衡量计算结果后者测量结果与真实结果之间的差值的参考量值。也就是用来衡量获得值与真相值之间的差距。

误差越大，说明实际结果或者实验结果偏离真实结果或者真相结果的程度越大。
误差越小，说明实际结果或者实验结果偏离真实结果或者真相结果的程度越小。

在进行机器学习的模型的训练的时候，我们追求的是尽可能小的误差。

在我们使用机器学习获取结果时，一般有两种结果，一是正确结果，二是错误结果。一般情况下，正确结果是我们想要的目的结果。 我们一般使用正确率来表示获得结果与正确的目的结果之间的误差。用错误率来表示获得结果与错误的目的结果之间的误差。

机器学习的正确率越大，错误率越小，说明该模型的可靠性越大，准确性越大，模型的性能越好。


训练误差 (training error)：
也称为 经验误差 (empirical error)，描述的是学习器相对于某一特定的训练集的误差。

泛化误差 (generalization error)：
描述的是将训练得到的模型应用于性能测试的时候，应用于测试集的时候，模型反映出来的误差。

一般来说，在进行模型训练的时候，训练的学习模型是不知道或者不接触用于测试的新样本的。所以在训练的时候，应该竟可能的使得模型的训练误差小。

也就是说，在训练模型时，能够控制的只有训练误差，通过尽量减少训练误差来实现尽可能减少泛化误差的目的。


1.2 拟合

在进行机器学习的时候，我们通过设计学习算法对训练数据的规律特征进行学习训练从而生成具有一定计算或判断处理的模型。

我们训练学习模型的时候，是针对于训练数据特定的训练特征进行训练的，而我们训练的模型，是希望能够完好地提取并反映我们关注的特征。

然而，当我们使用现今的学习算法对训练数据的特征进行学习的时候，最终训练得到的模型学习了训练数据的很多特征。 并不是所有的模型最终学习的训练数据的特征都是我们想要的。

我们可以这样理解，我们训练学习模型是针对具体的某一特定情境或者某一特定问题，而训练数据通过不同的算法，能够被提取出的规律特征有很多种。 但是这些被学习的训练特征，有些是问题需要的，有些是问题不需要的。

我们将有用的训练特征称为 有用训练特征，将无用的训练特征称为 无用训练特征 。


有用训练特征 :
有用训练特征能够帮助研究者强健其所设计的学习模型，是训练的模型需要的。在设计学习模型时，研究者应该尽可能地将更多的有用训练特征考虑进去。


无用训练特征 :
无用训练特征不能帮助研究者强健其所设计的学习模型，不是训练的模型需要的。在设计学习模型时，研究者应该尽可能地避免将无用训练特征考虑加入。




拟合 :
拟合指的是机器学习算法从训练集中提取学习训练特征的从而生成能够反映一定训练集特性的机器学习模型的过程。


适当拟合 :
适当拟合指的是学习算法从训练集中学习提取适当数量与种类的训练特征而训练生成机器学习模型的过程。

这里这里的适当指的是被提取的训练特征的数量与种类不过多或者过少，数量及种类能够满足训练生成的机器学习模型的需要。


过拟合 :

过拟合指的是学习算法从训练集中学习提取过多的数量与种类的训练特征，导致许多无用的训练特征被考虑和加入模型中，使得训练生成机器学习模型具有强泛化能力（过度的泛化能力）的现象。


欠拟合 :

欠拟合指的是学习算法从训练集中学习提取太少的数量与种类的训练特征，导致许多有用的训练特征没有被考虑和加入模型中，使得训练生成机器学习模型具有弱泛化能力（欠少的泛化能力）的现象。


在进行实际的机器学习的时候，设计者希望得到的是恰当拟合，而不是过度学习训练特征的过拟合或者欠少学习训练特征的欠拟合。我们可以结合上一图加深对恰当拟合、过拟合及欠拟合的理解。




2. 评价学习模型性能优劣的方法

通常情况下，我们使用测试集（测试样本集合）来测试已经训练得到的学习模型的学习能力及模型性能。

收集测试集时，应该尽量使得测试样本与之前使用的，用于训练待测试的学习模型的训练样本集合中的各个样本都不同。

测试集与训练集差异性越大，测试误差就越真实，得到的测试结果就越准确。

所以，在进行机器学习的时候，有一个关键的前提步骤是，将数据集合理地划分成训练集与测试集。下面介绍几种重要方法。


2.1 留出法 (hold-out)

留出法是英译过来的，“留”就是 hold ，“出” 就是 out 。
留出法指的是直接将整体的数据集划分成两部分，一部分数据保留作为训练集，剩余的另一部分划分出去作为测试集。

一般在使用留出法划分训练集与测试集的时候，使用者需要制定一个划分比例。
假设整体的数据集的数量为 D，需要的训练集的数量为 P，需要的测试集的数量为 T 。
我们将训练集占整体数据集的比例称为 rp ，将测试集占整体数据集的比例称为 rt ， 有：

rp + rt = 1



按照集合的思想，则有以下关系：
P ∈ D ，  T ∈ D

P ∩ T = ∅





2.2 交叉验证法 (cross validation)

交叉验证法的执行原理是，将数据集D划分成n个大小相似的子集：

D = D1 ∪ D2 ∪ D3 ... ∪ Dn

Di ∪ Dj = ∅


被划分的数据集的子集互斥，子集与子集之间没有相互包含的交集部分。

下面本文将结合下图对交叉验证法的具体执行原理进行详细的阐述：
1. 首先，将整体数据集近似平均地分成 n 份；
2. 每次训练模型前，从 n 份数据子集中提取一份数据子集作为测试集，剩余的 n-1 份数据子集作为训练集；
3. 每次训练模型后，用之前提取的测试集对生成的模型进行测试，记录测试结果为 R1 ；
4. 重复第2步与第3步，直至刚开始划分的 n 份数据子集都充当了一次测试集；
5. 得到 n 个测试结果 ：R1, R2, R3, ... , Rn ；
6. 最后对获得的 n 的测试结果进行算术平均处理，得到最终的平均结果 Raverage 为：

Raverage = (R1 + R2 + R3 + ... + Rn)/n
