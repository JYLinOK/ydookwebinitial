可变数量参数 varargs
与其他的高级语言类似的是，Kotlin 在追求简便表达的道路上加入了 可变数量参数 的概念。

Kotlin 的可变数量参数的定义使用关键字 varargs 进行标志。

varargs 可变数量参数的加入使得了 Kotlin 函数在参数输入的时候更加的方便与灵活。



使用关键字 varargs 定义
Kotlin 函数参数的最后一个参数或者第一个参数可以被vararg标记。标记后的参数会变成array类型，类似于 java 中的 int...可变参数的用法。

spread 伸展操作符 *
在使用 Kotlin 的 vararg 定义的可变数量参数后，在以后调用该函数时，可以直接项函数中依次单个传入所需数据类型的参数，也可以单次传一个数组。

向可变数量参数传值一个数值的时候，需要保证传入的数组与可变数量参数所要求的数据类型是一致的。

向可变数量参数传值一个数值的时候，需要使用 * 标志传入的数组参数变量，以表明该参数代表的是具有多个参数的容器数组。

infix 中缀表示法
Kotlin 引入的中缀表示法是一种简化书写的函数调用表示法。

对 Kotlin 中某些特定性质的函数，使用 infix 中缀表示法可以简化函数调用的表达，增加代码的可读性。

并不是所有的 Kotlin 函数都适合使用 infix 中缀表示法。Kotlin 函数要使用中缀表示法，必须满足下面条件：
1. 函数必须是成员函数或扩展函数；
2. 函数只能有一个输入参数；
3. 函数唯一的输入参数不能是 可变数量参数 也不能是 默认参数。



结合关键字 infix ，使用中缀表达式定义声明函数
使用标志符 infix 定义中缀表达式，位置写在 fun 关键字前面。

中缀表达式的优先级低于算术操作符、 类型转换操作符以及 rangeTo 操作符。

中缀表达式需要指定调用者与被调用参数
中缀表达式必须需要指定调用者与被调用参数。

函数作用域
理解函数作用域是掌握函数的基本。

Kotlin 函数声明可以采用以下方式：
1. 在源文件顶层声明；
2. 在局部作用域声明；
3. 作为成员函数声明；
4. 作为拓展函数声明。

也就是说，Kotlin 函数可以定义为：
1. 源文件顶层函数；
2. 局部作用域内的函数；
3. 成员函数；
4. 拓展函数



局部函数
局部函数就是定义在局部作用域中的函数，通常是定义在类及函数的内部。

局部函数的作用范围只在局部作用域中有效。

Kotlin 支持直接嵌套使用的内部函数声明及定义。

局部函数访问外部函数的成员函数及成员参数
局部函数对于其所在的作用域以及该作用域往上的外包作用域的成员函数和成员参数具有访问及调用功能。

局部函数可以访问外部函数的成员函数及成员参数。称为：内部函数可以访问调用外部函数成员。

外部函数可以访问局部函数的成员函数及成员参数。称为：外部函数不能访问调用内部函数成员。

外部函数也称为 闭包。

尾递归函数
Kotlin 支持一种称为尾递归的函数式编程风格。

尾递归书写风格允许将一些通常用循环算法改写成递归函数，并且改写是而无堆栈溢出的风险。

在 Kotlin 中使用尾递归书写风格改写循环函数，只需要在待改写的函数声明的 fun 关键字的加上修饰标志符 tailrec 。

使用 tailrec 修饰循环函数后，Kotlin 编译器会自动地优化循环函数中的递归算法，使得原算法变得更加高效。

使用代码示例：

1 
tailrec fun recursionFun() { ... }
